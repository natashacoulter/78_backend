# RESTful API 
RESTful API - архитектурный стиль взаимодействия между приложениями
Он основан на протоколе HTTP. То есть описание правил обмена данными между клиентом
и сервером. 

1. Resources
 - URL - Uniform Resource Locator (путь по которому находится необходимый ресурс)

Используются существительные (в ед. или множ. числе)
Не использовать глаголы!

/users - получить данные всех пользователей
/users/629 - получить данные пользователя по id или 
             для удаления пользователя с id (629) в 
             зависимости от метода

Примеры плохих названий
/getAllUsers
/users/18/posts/14 (излишняя вложенность)

Самый лучший способ
/user-profile (kebab case)

Переменные пути
/users/:id

Query params (парметры запроса)
url?key=value&key=value (/products?size=32&color=red)

2. Клиент - сервер
METHODS
CRUD

CREATE - POST - создать пользователя (/users)
READ - GET - получить всех пользователей (/users)
UPDATE - PUT - отредактировать данные пользователя (/user/:id)
         PATCH - частичное редактирование данных (/user/:id)
DELETE - DELETE - удалить пользователя (/user/:id)

STATUS CODE
100 - 199 - info
200 - 299 - success (201 - created)
300 - 399 - redirect
400 - 499 - client error
500 - 599 - server error

3. Stateless - отсутствие состояния. Сервер не хранит данные о запросах, которые
были отправлены. Каждый запрос должен иметь необходимую информацию

4. Единообразие
Вся информация должна быть передана в общем формате - JSON
